# 儿童英文作文 AI 批改 — 产品需求

## 1. 实现目标

对一篇**孩子的英文作文**进行智能批改，支持从答题纸图片到识别、批改与结果展示的全流程。

## 2. 功能要求

### 2.1 **上传界面**

- 提供**界面**，支持用户**上传答题纸图片**（作文答题纸的扫描/拍照件）。

### 2.2 批改处理（大模型 + 相关能力）

上传成功后，由系统调用大模型及相关能力完成批改，需满足以下子项。

#### 2.2.1 二维码信息识别

从答题纸上的**二维码**中识别内容。二维码表示为**一个字符串**，各业务字段按固定顺序用 **连字符 `-` 拼接**（示例格式如下，以实际制码规则为准）：

```text
课程ID-班级ID-排课ID-学号-encodeURIComponent(学生姓名)-性别
```

- **学生姓名与中文支持**：第 5 段为「学生姓名」的**编码形式**。制码时与 Web 端一致，对姓名字符串使用 **`encodeURIComponent`** 编码（例如中文姓名会变为 `%E5%BC%A0...` 这类百分号转义形式），再作为整段内容写入二维码字符串；识别解析时对该段执行 **`decodeURIComponent`**，还原为显示与业务使用的**原始姓名**（含中文、emoji 等 `encodeURIComponent` 能表示的字符）。

识别后需解析出以下字段，顺序与字符串分段一一对应：

| 顺序 | 信息项   | 说明     |
| ---- | -------- | -------- |
| 1    | 课程 ID  | 课程标识 |
| 2    | 班级 ID  | 班级标识 |
| 3    | 排课 ID  | 排课标识 |
| 4    | 学号     | 学生学号 |
| 5    | 学生姓名 | 经 `encodeURIComponent` 编码后置于码串；解析时 `decodeURIComponent` 还原，以支持中文等字符 |
| 6    | 性别     | 学生性别 |

除姓名外的字段若与 `-` 分段规则产生歧义，应在制码/解析规范中另行约定转义或分段方式。

#### 2.2.2 手写内容 OCR 与去模板

- 对学生**手写作文**进行 **OCR** 识别，得到可编辑的文本。
- **剔除**答题纸上**预印的模板/固定说明文字**（只保留或突出学生**手写**的有效作文内容）。

#### 2.2.3 AI 作文批改

- 在去除模板后的**手写作文正文**上，由 AI 进行批改：
  - 识别**语法错误**；
  - 对每一处问题给出**修正后的正确语句**（可对应原句或片段级修正，具体粒度可在实现/交互上约定）。

#### 2.2.4 AI 作文评分

对学生作文从以下 **4 个维度**进行评分，各维度评分由指定的大模型负责：

| 维度         | 评分依据说明                                                   | 评分模型           |
| ------------ | -------------------------------------------------------------- | ------------------ |
| 卷面整洁     | 根据 OCR 识别结果判断书写工整度、涂改情况及排版整洁程度       | OCR 识别大模型     |
| 内容要点     | 判断作文内容是否切题、要点是否完整、论述是否充分               | 作文批改大模型     |
| 语言质量     | 评估词汇丰富度、语法准确性及表达流畅程度                       | 作文批改大模型     |
| 篇章结构     | 评估段落划分是否合理、逻辑层次是否清晰、开头结尾是否完整       | 作文批改大模型     |

- 每个维度输出一个**分值**（具体分制可在实现中约定，如 0–10 分或 0–25 分）及对应的**简短评语**。
- 各维度分值可叠加为**综合得分**，便于整体呈现。

## 3. 批改结果展示

批改完成后，在**同一界面**（或结果页）向用户展示，至少包括：

- **二维码信息**：展示**二维码识别与解析**得到的全部内容，与 [2.2.1](#221-二维码信息识别) 中字段一致，即**课程 ID、班级 ID、排课 ID、学号、学生姓名**（已 `decodeURIComponent` 还原后的可读姓名）、**性别**，便于核对学生身份与课次信息。

| 展示项                 | 说明                                                         |
| ---------------------- | ------------------------------------------------------------ |
| 二维码信息识别内容     | 解析后的课程/班级/排课/学号/姓名/性别                        |
| OCR 手写原文           | 经识别并去除模板后保留的学生作文文本                         |
| 语法错误列表           | AI 标出的语法问题列表                                        |
| 修正后的正确语句列表   | 与错误对应的修正建议/正确表达                                |
| 各维度评分与评语       | 卷面整洁、内容要点、语言质量、篇章结构各维度的分值及简短评语 |
| 综合得分               | 各维度分值汇总，直观呈现学生整体表现                         |

## 4. 说明与边界（可选扩展）

- 本需求以「答题纸 + 二维码 + 手写作文」为典型场景；若存在无二维码、纯线上录入等变体，可另列补充需求。
- 「剔除模板」与「手写/印刷区分」的准确度依赖 OCR 与规则/模型能力，宜在验收标准中明确可接受误差的范围。
