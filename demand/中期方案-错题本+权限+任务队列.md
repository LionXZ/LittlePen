## 1. 错题本

### 1.1 需求

自动汇总学生所有作业中的错误，按错误类型分组展示，帮助老师和学生针对薄弱环节进行专项练习。

### 1.2 数据模型

新增 `error_book` 表，同步于每次批改完成时写入：

```sql
CREATE TABLE error_book (
  id          BIGINT PRIMARY KEY AUTO_INCREMENT,
  student_id  VARCHAR(64)  NOT NULL,  -- 学号
  subject     VARCHAR(16)  DEFAULT 'en',  -- 科目
  error_type  VARCHAR(32)  NOT NULL,  -- 语法/拼写/用词/标点
  original    TEXT,                   -- 错误原文
  corrected   TEXT,                   -- 修正建议
  explanation TEXT,                   -- 中文解释
  record_id   VARCHAR(36),            -- 关联的批改记录
  created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_student (student_id),
  INDEX idx_subject (subject)
);
```

写入时机：`batch_processor.py` 的 `update_record_status(record_id, status=2, result)` 成功后，从 `result.grammar_errors` 中逐条写入。

### 1.3 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/error-book/{student_id}` | 学生错题列表（支持 `?subject=en` 筛选） |
| `GET` | `/api/v1/error-book/{student_id}/stats` | 错误类型分布统计 |

响应示例：

```json
// GET /error-book/S001?subject=en
{
  "items": [

// GET /error-book/S001/stats
{
  "total": 15,
  "by_type": { "语法": 8, "拼写": 4, "用词": 2, "标点": 1 },

```

## 2. 权限体系

### 2.1 需求

- 学生只看自己的作业和错题
- 老师看自己班级所有学生的作业
- 管理员看全部

### 2.2 数据模型

```sql
CREATE TABLE users (
  id            BIGINT PRIMARY KEY AUTO_INCREMENT,
  username      VARCHAR(64) UNIQUE NOT NULL,
  password_hash VARCHAR(256) NOT NULL,
  role          VARCHAR(16) NOT NULL DEFAULT 'student',  -- student/teacher/admin
  display_name  VARCHAR(64),                             -- 显示名称
  class_id      VARCHAR(64),                             -- 关联班级（老师为空）
  created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3 认证方案

**JWT**，不使用 OAuth/第三方登录。

```
POST /api/v1/auth/login   { username, password } → { access_token }
POST /api/v1/auth/register { username, password, role, class_id } → { user }
```

### 2.4 权限中间件

```python
# backend/src/api/auth.py
async def get_current_user(token: str = Depends(oauth2_scheme)) -> dict:
    payload = jwt.decode(token, SECRET_KEY)
    user = await get_user_by_id(payload["user_id"])
    return user

def require_role(*roles: str):
    """角色校验装饰器"""
    ...
```

### 2.5 现有接口权限改造

```
GET /essays          → student 只能看自己的, teacher 看班级的, admin 看全部
GET /essay/{id}      → 校验: 只能看自己的/自己班级的
GET /error-book/{id} → 同上
GET /stats/*         → teacher 只能看自己班级的
POST /batch/upload   → teacher/admin 才有权限（学生不能上传）
```

### 2.6 前端

| 文件 | 改动 |
|------|------|
| `views/login/index.vue` | **新文件** — 登录页 |
| `router/index.ts` | 路由守卫 `beforeEach`：未登录跳 `/login` |
| `api/essay.ts` | axios 拦截器注入 `Authorization: Bearer xxx` |
| `App.vue` | 侧边栏底部显示当前用户名 + 退出按钮 |

### 2.7 不做什么

- 不做注册页（管理员后台手动创建账号，或命令行批量导入）
- 不做 RBAC 细粒度权限
- 不做密码找回

## 3. 异步任务队列

### 3.1 现状

```
asyncio.create_task(process_batch(batch_id))
  → 进程内运行, 服务重启任务丢失 → recover_unfinished() 补救
  → Semaphore(3) 控制并发
```

### 3.2 目标

```
Celery task → Redis broker → Celery worker 执行
  → 任务持久化到 Redis, 重启不丢
  → 优先级队列 (老师提交的 > 学生提交的)
  → 失败自动重试 (max_retries=3)
  → Flower 监控面板
```

### 3.3 触发时机

**不是现在**。当前 `asyncio + Semaphore + recover_unfinished` 已经够用。当出现以下信号时再迁移：

- 单日批改请求超过 500 次
- 频繁出现"服务重启导致未完成任务需要恢复"的告警
- 需要多 worker 分布式处理

### 3.4 实现（预留设计）

```python
# backend/src/pipeline/tasks.py
from celery import Celery

app = Celery('littlepen', broker='redis://localhost:6379/0')

@app.task(bind=True, max_retries=3, default_retry_delay=60)
def grade_single_task(self, record_id: str, image_path: str):
    try:
        result = process_batch_item(record_id, image_path)
        return result
    except Exception as e:
        self.retry(exc=e)
```

```python
# routes.py 中
from src.pipeline.tasks import grade_single_task

# 原来: asyncio.create_task(process_batch(batch_id))
# 改为:
grade_single_task.delay(record_id, image_path)
```
